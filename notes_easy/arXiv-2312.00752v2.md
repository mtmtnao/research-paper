# Mamba: Linear-Time Sequence Modeling with Selective State Spaces（選んで覚える「マンバ」: 文の長さに比例した時間で動く新しい言語モデル）

- arXiv: https://arxiv.org/abs/2312.00752
- 一次ソース: ../papers/arXiv-2312.00752v2/
- 正規ノート: ../notes/arXiv-2312.00752v2.md

---

## 一言で言うと

「いま入ってきた言葉に合わせて、覚えるか忘れるかを切り替えられる」新しいタイプのニューラルネットワークで、Transformer 並みの賢さを文の長さに比例した速さで実現した話。

## どんな問題を解こうとしてるの？

- 今の AI（ChatGPT みたいなやつ）は **Transformer**（← 文を理解するための AI の設計図のこと。今ほとんどの言語 AI がこれを使ってる）でできている。これは「文中の全部の単語どうしを総当たりで見比べる」仕組みなので、すごく賢いけど **文が長くなると計算量が二次関数的に増える**。
  - たとえば文が 10 倍長くなると、計算は 100 倍になる。1000 文字の小説を読ませるのと、100 万文字の本一冊を読ませるのとでは、必要な計算がとんでもないことになる。
- 一方、もっと軽い方法として **SSM（State Space Model、状態空間モデル）** という仲間がいる。これは「直前までの内容を 1 つの小さな『メモ帳』にまとめておく」やり方で、文の長さに **比例** した時間で済む。
- でも従来の SSM には致命的な弱点があった: **メモ帳の更新ルールが、入ってくる単語によらず常に同じだった**（これを LTI = Linear Time Invariant、「いつでも同じ動き方をする」性質と呼ぶ）。
  - これだと「あ、いまの単語は大事だから覚えておこう」「これは『えーっと』みたいなどうでもいい単語だから無視しよう」みたいな **内容に応じた判断ができない**。
  - 結果、音声や画像のような「ずっと連続的に流れる」データには強いけど、言葉や DNA のような「重要度がバラバラな」データに弱かった。

身近な例えで言うと、授業中にノートを取るときに「先生がいま言ったことを全部そのまま写す」ロボットみたいなもの。重要な公式と「えーっと」みたいな口癖を区別できない。

## どうやって解いたの？

### コアアイデア

**「メモ帳の更新ルール自体を、いま入ってきた単語に合わせて毎回切り替える」** という発想。これが論文のキーワード **selectivity（選択性）** だ。

例え話: 普通の SSM のメモ帳ロボットは「3 秒ごとに今聞こえた言葉を必ずノートに書く」と決まっているロボット。Mamba のロボットは「いま聞こえたのが大事そうな言葉なら大きく書き、どうでもいい言葉ならスキップする」と毎回判断できるロボットになった。判断のしかた自体も学習で覚える。

### 仕組み

- step1: 普通の SSM では「メモ帳をどれくらい更新するか」を決めるツマミ（$\Delta, B, C$ という 3 つのパラメータ）が **固定** だった。Mamba ではこのツマミを **そのとき入ってきた単語の値から毎回作り直す**（入力依存にする）。
- step2: ツマミが毎ステップ変わるので、昔の SSM みたいに「掛け算の表をあらかじめ作っておいて一発で計算」（convolution、畳み込み）ができなくなる。その代わり **parallel scan（並列スキャン）** という、足し算を並列で順次計算できる賢い計算法を使う。
- step3: GPU（計算用の半導体）の中には「速いけど狭いメモリ（SRAM）」と「遅いけど広いメモリ（HBM）」がある。Mamba はメモ帳の中身を「速い狭いメモリ」だけで完結させて、「遅い広いメモリ」に書き出さない。これで FlashAttention というテクニックと同じ発想で **桁違いに速く** なる（標準実装より 20〜40 倍速い）。
- step4: ネットワーク全体の設計図もシンプルにする。普通の Transformer は「注意機構 (attention) のブロック」と「MLP（多層パーセプトロン、つまり普通の計算層）のブロック」を交互に積み重ねる。Mamba は両方を 1 つにまとめた **同じ形のブロック** を縦に積むだけ。

### 主要な数式

#### ① SSM の基本ルール（メモ帳の更新）

$$ h_t = \overline{A}\, h_{t-1} + \overline{B}\, x_t, \qquad y_t = C\, h_t $$

**この式が言ってること**: 「いまのメモ帳 $h_t$ は、ひとつ前のメモ帳 $h_{t-1}$ をちょっと変化させたものに、新しく入ってきた言葉 $x_t$ を混ぜたもの」。出力 $y_t$ はそのメモ帳から取り出した答え。これを 1 単語ずつ繰り返していく。

**記号の意味**:
- $x_t$ … 時刻 $t$ に入ってきた言葉（数字に直したもの）
- $h_t$ … 時刻 $t$ でのメモ帳の中身（複数の数の組）
- $y_t$ … 時刻 $t$ にモデルが出す答え（数字）
- $\overline{A}$ … 「前のメモ帳をどう古くしていくか」を決める係数（古い記憶をどれくらい残すか）
- $\overline{B}$ … 「新しい言葉をどれくらいメモ帳に書き込むか」を決める係数
- $C$ … 「メモ帳のどの部分を答えとして読み出すか」を決める係数

**身近な例え**: 連絡帳に毎日メモを足していくのと同じ。$\overline{A}$ は「昨日までのメモを 0.9 倍に薄める」みたいな割合、$\overline{B}$ は「今日の出来事をどれくらいの濃さで書くか」、$C$ は「明日テストで聞かれそうな部分を抜き出す」マーカーペン。

---

#### ② Mamba の肝: ツマミ $\Delta, B, C$ を入力で作り直す

$$ B_t = \mathrm{Linear}(x_t), \qquad C_t = \mathrm{Linear}(x_t), \qquad \Delta_t = \mathrm{softplus}(\mathrm{Linear}(x_t)) $$

**この式が言ってること**: ①の式に出てきた $B, C, \Delta$ の 3 つのツマミを、**「いま入ってきた単語 $x_t$ から計算して毎回作り直す」**。つまりツマミが時刻ごとに違う値になる。

**記号の意味**:
- $x_t$ … いま入ってきた単語
- $\mathrm{Linear}(\cdot)$ … 「入った数に何かをかけて足す」だけのシンプルな変換器（学習で覚える）。中学校で習う $y = ax + b$ みたいなやつを、複数の数についてまとめてやるもの
- $B_t, C_t, \Delta_t$ … 時刻 $t$ のときのツマミ（時刻ごとに変わる）
- $\mathrm{softplus}$ … 入った値がマイナスでも必ずプラスに変える関数（$\Delta$ は「どれくらい時間を進めるか」だから負だと困る）

**身近な例え**: コーヒーを淹れる人が、お客さんごとに「この人は甘党だから砂糖多め」「この人は苦いの好きだから濃いめ」と毎回レシピを変えるのと同じ。普通の SSM は「全員に同じレシピ」だった。Mamba は「お客さんの注文を聞いてからレシピを決める」。

---

#### ③ ゲート（門）との関係 — Theorem 3.1

$$ g_t = \sigma(\mathrm{Linear}(x_t)), \qquad h_t = (1 - g_t)\, h_{t-1} + g_t\, x_t $$

**この式が言ってること**: 特別な設定（メモ帳のサイズが 1 のとき）では、Mamba の更新式は「**今の単語をどれくらい採用するか**」を決めるツマミ $g_t$ を使った「混ぜ算」になる。$g_t$ が 1 に近ければ「新しい単語をそのまま採用」、0 に近ければ「前のメモ帳をそのまま残す」。

**記号の意味**:
- $g_t$ … 「ゲート」と呼ばれる 0〜1 の数（門の開き具合）
- $\sigma$ … シグモイド関数。どんな数を入れても 0〜1 の間に押し込めてくれる関数。要するに「割合」に変える道具
- $h_{t-1}$ … ひとつ前のメモ帳
- $x_t$ … 今の単語

**身近な例え**: 水道の蛇口の混合栓。$g_t$ が「お湯の割合」だと思えばいい。$g_t = 1$ なら全部お湯（新しい単語そのまま）、$g_t = 0$ なら全部水（古いメモ帳そのまま）、$g_t = 0.3$ なら 3 割お湯・7 割水を混ぜる。お客さんごとに（=単語ごとに）この蛇口の開き具合を変えられるようになった、というのが Mamba の本質。

> 補足: 「Theorem 3.1（定理 3.1）」は論文中の証明付きの主張で、「昔から RNN（リカレントニューラルネットワーク、過去の入力を覚えながら処理するタイプの古典的なネットワーク）で使われてきた『ゲート』という仕組みは、実は SSM を離散化したときに自然に出てくるものなんだ」と理論的につないでいる。つまり「マンバの選択性 = 古き良き LSTM/GRU のゲートを SSM の言葉で再発明したもの」と読める。

## 何がすごいの？

数値は全部 TeX から拾った数字だよ。

- **合成タスク Selective Copying**（決まった文字を選んで覚えてオウム返しするテスト）: 旧来の S4 は 18.3%（ほぼ偶然） → Mamba は **99.8%** で完全に解けた。
- **合成タスク Induction Heads**（パターンを覚えてもう一度出てきたときに次を予測するテスト）: 長さ 256 で学習させただけなのに、長さ $2^{20}$ = **約 100 万トークンまで完璧に外挿** できた（学習時の 4000 倍の長さ！）。他の手法はせいぜい 2 倍までしか伸びない。
- **言語モデルの性能（Pile というデータセット、6 つの常識タスク平均）**:
  - Mamba-1.4B: **平均 59.7** vs Pythia-1.4B: 55.2 / RWKV-1.5B: 54.3
  - Mamba-2.8B: **平均 63.3** vs Pythia-2.8B: 59.1 / RWKV-3B: 59.6
  - Mamba-2.8B は **2 倍以上大きい GPT-J-6B（63.0）と並ぶ**。Pile の perplexity（言語モデルの予測の苦しさ指標、低いほどよい）も Mamba-2.8B が 6.22 で Pythia-2.8B の 6.73 を上回る。
- **DNA モデリング**（HG38 という人ゲノムデータセット）: Transformer や HyenaDNA と **同じ精度を 3〜4 倍少ないパラメータで** 出せる。さらに「文脈を長くするほど精度が単調に良くなる」性質を実証（HyenaDNA は逆に悪くなる）。
- **音声生成（SC09 という 0〜9 の数字音声データセット）**: たった 6.1M パラメータの Mamba で FID（生成品質指標、低いほどよい）= **0.94**。これは比較対象の SaShiMi (5.8M) の **1.99 を半分以下に**。24.3M モデルでは FID = **0.67** で SOTA（state-of-the-art、その時点の最高記録）。
- **速度**: 学習時の selective scan は標準 PyTorch 実装より **20〜40 倍速い**。推論（文の生成）スループットは同サイズ Transformer の **5 倍**。Mamba-6.9B のほうが Transformer-1.3B より生成が速いというくらい有利。

**著者が論文で主張している貢献**:
1. SSM のパラメータを入力依存にする「選択メカニズム」を提案
2. ハードウェアを意識した並列スキャンで、選択性を入れても実用速度を保つ実装を提案
3. attention も MLP もない、シンプルで均質な単一ブロック構造 **Mamba** を提案
4. 言語・DNA・音声の 3 つの異なる種類のデータで Transformer 同等以上を線形時間で達成

## キーワード辞典

- **Transformer** … 今の主流 AI（ChatGPT など）の中身に使われている設計図。文中の全単語どうしを総当たりで見比べる「attention」が中核。文が長くなると計算量が二次で増える。
- **attention（注意機構）** … 文中の「どの単語に注目するか」を計算する仕組み。Transformer の心臓部。
- **SSM（State Space Model、状態空間モデル）** … 入力を 1 つの小さなメモ帳（隠れ状態 $h$）にまとめながら順番に処理する仕組み。文の長さに比例した時間で動く。
- **S4** … SSM の代表格の 1 つ。Mamba の元になった研究。
- **S6** … S4 に Mamba 流の選択メカニズムを足したもの。S4 + selection + scan の頭文字。
- **LTI（Linear Time Invariant）** … 「いつでも同じ動き方をする」性質。従来 SSM の弱点。
- **recurrence（リカレンス／再帰）** … 「ひとつ前の状態から今の状態を作る」式 1 行ずつの計算。
- **convolution（畳み込み）** … 1 つの「型紙」を端から端まで掛け算して足し合わせる計算。画像処理でもおなじみ。
- **RNN** … Recurrent Neural Network。過去の情報をメモしながら順に処理する古典的ネットワーク。LSTM/GRU はこの仲間。
- **gate（ゲート）** … 「新情報をどれくらい採用するか」を 0〜1 で決める門。LSTM/GRU の心臓部。
- **HBM / SRAM** … GPU の中のメモリ。HBM は広いけど遅い、SRAM は狭いけど速い。
- **FlashAttention** … attention を SRAM 内で完結させて速くするテクニック。Mamba の scan 実装もこの発想をマネしている。
- **parallel scan（並列スキャン）** … 「足し算を順番にやる」処理を、賢く並列化するアルゴリズム。
- **Pile** … 800GB 級の英語コーパス。言語モデルの標準的な事前学習データセット。
- **perplexity（ppl）** … 言語モデルが次の単語を予測するときの「迷い具合」。低いほど良い。
- **zero-shot** … 追加学習なしで、新しいタスクをいきなり解かせる評価方式。
- **HellaSwag / PIQA / Arc-E / Arc-C / WinoGrande / LAMBADA** … 言語モデルの常識・推論能力を測る標準ベンチマーク 6 種。
- **Pythia / RWKV / GPT-J / OPT** … 比較対象になっているオープンソース言語モデルたち。
- **FID / IS** … 音声や画像の生成品質指標。FID は低いほど良く、IS は高いほど良い。
- **SaShiMi / HyenaDNA / Hyena / RetNet** … 比較対象の系列モデルたち。SaShiMi は音声、HyenaDNA は DNA、Hyena は畳み込み系、RetNet は recurrent 系。
- **foundation model** … 大量データで事前学習した「土台」モデル。下流タスクに使い回せる。
- **softplus** … マイナスの数も含めて、なめらかにプラスに変える関数。
- **sigmoid（$\sigma$）** … どんな数も 0〜1 に押し込める関数。割合を表すのに便利。

## ちょっと深掘り（中学生は飛ばして OK）

- **なぜ $\Delta$（時間刻み）が一番大事なのか**: 離散化の式 $\overline{A} = \exp(\Delta A)$ から、$A$ への入力依存性は $\Delta$ 経由でも作れる。だから「$A$ も入力依存にする」必要はなく、$\Delta, B, C$ だけ入力依存にすれば十分、というのが著者の判断。Ablation でも選択 $\Delta$ 単独で perplexity が 10.93→9.81 まで改善し、$B, C$ も足すと 8.71 まで下がる。
- **なぜ scan を実装し直したか**: 入力依存になると「型紙を 1 個用意して全位置に当てる」畳み込みの形が使えなくなる。一方 recurrence は順次計算で並列化しづらい。著者は parallel scan + kernel fusion + recomputation という古典テクの組み合わせで「状態 $h$ のサイズが (B, L, D, N) と大きくても、それを HBM に書き出さず SRAM 内で計算完結させる」工夫をした。これで実用速度を確保。
- **Real vs Complex**: 従来 SSM は $A$ を複素数で持つことが多かったが、Mamba は実数をデフォルトに。350M モデルの ablation で S4D-Real ($A_n = -(n+1)$) が ppl 8.71 で S4D-Lin (複素 $A_n = -1/2 + ni$) の 9.16 を上回る。ただし著者は「audio waveform 実験だけは複素に戻した」と注記しており、データの性質によって最適は変わる（"no free lunch"）。
- **論文が認める限界**:
  - scaling は約 1.3B、zero-shot 評価は 2.8B 止まりで、7B+ スケールでの検証は本論文ではされていない（著者自身が discussion で言及）。
  - fine-tuning や RLHF、量子化など実用化に必要な手順が Transformer と同じように乗るかは open question として残している。
  - 連続信号（音声波形など）では LTI の方が良い場面がある、と明記している。
- **アーキは凝るより selection を入れる方が効く**: H3 アーキ + S6 が 99.7%、Mamba アーキ + S6 が 99.8%。つまり Mamba ブロックの形そのものの貢献より、内部の selective SSM (S6) を入れることの貢献の方が大きい、というのが著者の結論。
