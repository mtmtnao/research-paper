# ReAct: Reasoning（考えること）と Acting（行動すること）を AI に同時にやらせる方法

- arXiv: https://arxiv.org/abs/2210.03629
- 一次ソース: ../papers/arXiv-2210.03629v3/
- 正規ノート: ../notes/arXiv-2210.03629v3.md

---

## 一言で言うと

AI（言葉を扱うコンピュータ）に「**考えていることを言葉で書きながら、同時に道具を使って調べたり動いたりさせる**」だけで、推論も行動もうまくなる、という発見。

## どんな問題を解こうとしてるの？

- **大規模言語モデル（LLM）** ← 大量の文章を読んで学習した、文章を作るのが上手なコンピュータプログラム（ChatGPT のもと）のこと。これに「複雑な問題」を解かせたい。
- 例えば「ホグワーツに通ったハリー・ポッターの作者の出身地は？」みたいに、複数のステップで調べないと答えが出ない質問。あるいは「キッチンで机の上のランプの下にある紙を調べて」みたいに、家の中を動き回って実行しないといけない指示。
- これまでの研究は **2 つの方向にバラバラに進んでいた**:
  - **Chain-of-Thought（CoT、考えの連鎖）** ← LLM に「順番に考えを書き出してから答えて」と頼む方法。例：「まず A を思い出す。次に B。だから答えは C」。
    - 弱点：頭の中だけで考えるので、**ハルシネーション**（← AI が事実じゃないことを自信満々で言ってしまう現象）を起こしやすい。記憶違いを正す方法がない。
  - **Acting 系**（行動だけする方法） ← LLM に「次に何をするか」だけを言わせて、Wikipedia 検索ボタンを押させたり、ロボットを動かしたりする方法。
    - 弱点：「なぜそれをするか」を考えていないので、ゴールを分解したり、途中で計画を直したりできない。
- 人間は料理しながら「あ、塩がない。じゃあ醤油で代用しよう」みたいに、**考えと行動を交互に切り替える**。これを AI でも自然にやらせたい、というのがこの論文の出発点。

## どうやって解いたの？

### コアアイデア

AI に「**行動**」を出力させる代わりに、**「考え」も行動の 1 種類として扱う**。「考え」は環境を変えない（検索ボタンを押したりしない）けど、自分の頭の中のメモが増える。

例え話：宝探しゲームで、プレイヤーが取れる行動が「北へ歩く / 南へ歩く / 宝箱を開ける」の 3 つしかなかったとする。これに **「メモ帳に独り言を書く」** という 4 つ目の行動を足すイメージ。「宝はたぶん井戸の近くだから、まず東に行こう」とメモに書くと、次の一手を選びやすくなる。井戸を見つけたら「井戸を覗いた → 空だった。じゃあ次は橋の下を疑おう」とまたメモる。**メモを書くこと自体がゲームを進めないけど、次の動きが賢くなる**。これを LLM にやらせるだけ。

### 仕組み

- step1: LLM にお手本（few-shot example、1〜6 個の見本）を見せる。お手本は「`Thought:` （考え）」「`Action:` （行動）」「`Observation:` （結果）」が交互に並んだ文章。
- step2: 解きたい問題を最後にくっつけて、LLM に続きを書かせる。
- step3: LLM が `Action: search[ハリー・ポッター]` のように行動を書いたら、本物の Wikipedia 検索を呼んで結果をもらう。
- step4: その結果を `Observation:` として続けて貼り付ける。
- step5: また LLM に「次の Thought か Action」を書かせる。
- step6: `Action: finish[答え]` が出たら終わり。

タスクが「推論メイン」（質問応答など）なら考えと行動を **密に**（毎回考えてから動く）、タスクが「動きメイン」（家の中を歩き回るゲーム）なら考えを **まばらに**（大事な分岐のときだけ考える）出させる。どっちのモードにするかは、お手本の書き方で決まる。

ベースの LLM は **PaLM-540B** ← Google が作った、文章を 5400 億個のつまみ（パラメータ）で生成する巨大プログラム。ここでは「凍結（frozen）」、つまり追加学習はせずに、お手本を見せるだけで使う。

### 主要な数式

論文には大きな数式はほとんど出てきません。代わりに「行動と考えをどう書くか」という記号の約束ごとが少しだけ書かれています。中核は次の 2 つだけです。

---

**式 1：もともとの「AI エージェント」のお仕事の書き方**

$$ \pi(a_t \mid c_t), \quad c_t = (o_1, a_1, o_2, a_2, \dots, o_{t-1}, a_{t-1}, o_t) $$

**この式が言ってること**: 「今までに見たもの（$o$）と今までにやった行動（$a$）を全部ならべて、それを材料にして、次に何をするか（$a_t$）を決めるよ」という意味。$\pi$ は「決め方のルール」のこと。

**記号の意味**:
- $t$ … 「今が何手目か」を表す数字（1 手目、2 手目、…）。
- $o_t$ … $t$ 手目に **環境から見えたもの**（observation の頭文字）。例：Wikipedia 検索の結果、目の前にある机の上の物。
- $a_t$ … $t$ 手目に **AI が選んだ行動**（action の頭文字）。例：`search[アインシュタイン]`。
- $c_t$ … $t$ 手目の **コンテキスト**（context、文脈）。これまでの観察と行動を全部つなげた長い文章。
- $\pi(a_t \mid c_t)$ … 「$c_t$ という状況のとき、$a_t$ を選ぶ確率」。「ルール」「方針」のこと。$\pi$ はギリシャ文字のパイ。

**身近な例え**: 将棋で、「今までの全部の指し手」を見ながら次の一手を決めるのと同じ。$c_t$ が「これまでの棋譜」、$a_t$ が「次の一手」、$\pi$ が「あなたの将棋の指し方の癖」。

---

**式 2：ReAct がやった拡張（考えも行動の 1 種類にする）**

$$ \hat{\mathcal{A}} = \mathcal{A} \cup \mathcal{L} $$

**この式が言ってること**: 「新しい行動の選択肢の集まり（$\hat{\mathcal{A}}$）は、もともとの行動の集まり（$\mathcal{A}$）に、**言葉で書ける独り言ぜんぶ**（$\mathcal{L}$）を足したものだよ」。つまり「考えごとを書くこと」も行動の 1 つとして数えるね、という宣言。

**記号の意味**:
- $\mathcal{A}$ … もともとできた行動の集まり（例：`search`, `lookup`, `finish` の 3 つ、など）。$\mathcal{A}$ は花文字の A。
- $\mathcal{L}$ … **言葉で書ける文章ぜんぶ**の集まり（language の頭文字）。例：「ハリー・ポッターの作者を調べよう」「今のところ手がかりがない」など、人間が日本語や英語で書ける文章はすべて含まれる。だから無限に広い。
- $\hat{\mathcal{A}}$ … 拡張した新しい行動の集まり。$\hat{\;}$（ハット、^ の記号）は「ちょっと別バージョン」という印。
- $\cup$ … 「足し合わせる（合併する）」という記号。集合の足し算。

**身近な例え**: ジャンケンが「グー・チョキ・パー」の 3 択だったところに、新ルールで「**何も出さずに『次はパーを出すぞ』と心の中でつぶやく**」という 4 つ目を足すイメージ。つぶやいても相手は何も変わらないけど、自分の作戦が整理される。

---

考えを書いたとき、コンテキストは次のように更新されます。

$$ c_{t+1} = (c_t, \hat{a}_t) $$

**この式が言ってること**: 「次の手のコンテキストは、今までのコンテキストに、今書いた考え（$\hat{a}_t$）をくっつけたものだよ」。考えを書いただけだから環境（Wikipedia や部屋）は何も変わらず、つまり新しい $o$（観察）は増えない。**メモ帳に追記されただけ**。

**記号の意味**:
- $c_{t+1}$ … 次の（$t+1$ 手目の）コンテキスト。
- $c_t$ … 今のコンテキスト。
- $\hat{a}_t$ … 今書いた **考え（thought）**。$\mathcal{L}$ に属する、言葉で書かれた独り言。

**身近な例え**: テスト中に問題用紙の余白に「この問題、たぶん三平方の定理だな」とメモした瞬間。先生も他の生徒も何も変わらないけど、自分の手元の情報は 1 行増えている。

## 何がすごいの？

論文では **4 つの違うタスク** で、いろんなベースライン（比較対象）と勝負しています。数字はすべて TeX 中の表からの正確な値です。

**1. HotpotQA（複数の Wikipedia ページをまたいで答える質問応答、EM = 完全一致正解率 %）**
- Standard（普通に答えさせる）: 28.7
- CoT（考えだけ）: 29.4
- CoT-SC（CoT を 21 回やって多数決）: 33.4
- Act（行動だけ）: 25.7
- **ReAct（考えと行動）**: 27.4 ← 単独では CoT に少し負ける
- CoT-SC → ReAct（CoT で自信なかったら ReAct に切り替え）: 34.2
- **ReAct → CoT-SC（ReAct で答えが出なかったら CoT-SC へ）**: **35.1**（最強）
- ※ ただし「人間が学習データで鍛えた専用システム」は 67.5。まだ大差あり。

**2. FEVER（Wikipedia をもとに「その主張は正しい？」を判定、正解率 %）**
- Standard 57.1 / CoT 56.3 / CoT-SC 60.4 / Act 58.9 / **ReAct 60.9** / **CoT-SC → ReAct 64.6（最強）**

**3. ALFWorld（テキストだけの家の中を動き回るゲーム、134 個のテストでの成功率 %）**
- BUTLER（10 万個の手本で訓練した模倣学習エージェント）: 37
- Act だけ（best-of-6、6 個のプロンプトのうちの最高）: 45
- ReAct-IM（考えを「今の状況の言い換え」だけに制限した版）: avg 48 / best-of-6 53
- **ReAct（avg 57 / best-of-6 71）** ← 一番悪い trial の 48 でも他の手法の best を超えている
- Act に対する相対的な伸びは 33〜90%、平均 62% の改善。

**4. WebShop（Amazon を模した買い物サイトで指示通りの商品を買う、500 個のテスト指示）**
- IL（人間 1,012 件のデモで模倣学習）: Score 59.9 / 成功率 SR 29.1
- IL+RL（さらに 10,587 件で強化学習）: 62.4 / 28.7
- Act: 62.3 / 30.1
- **ReAct**: **66.6 / 40.0** ← 1〜2 個の手本だけで IL+RL を SR で絶対値 10 点超え
- ただし人間の達人は 82.1 / 59.6 でまだ遠い。

**5. 失敗の中身を人間が分類した分析（HotpotQA 各 50 trajectory × 4 種類 = 200 件）**
- CoT の失敗の **56% がハルシネーション**（事実をでっち上げ）。
- ReAct のハルシネーション失敗は **0%**。代わりに「同じ考え/行動を繰り返すループ」が頻発（reasoning error 47%）、検索結果が空っぽで詰むのが 23%。

**6. 追加学習（finetuning）したときの逆転**
- お手本だけ（prompting）だと、小さいモデル（PaLM-8B / 62B）では ReAct は 4 手法中 **最弱**。情報を覚えきれないため。
- でも 3,000 件の trajectory で追加学習すると **最強**に転じる。PaLM-8B で追加学習した ReAct は PaLM-62B の prompting すべてに勝ち、PaLM-62B 追加学習版は 540B の prompting すべてに勝つ。

**著者が「貢献（contribution）」として挙げているのは 4 つ**:
1. 推論と行動を **交互に出す** という汎用プロンプト手法 ReAct を提案。
2. 4 種類の違うタスクで他手法と比較して有効性を示した。
3. 「推論タスクに行動はどれくらい効くか」「決定タスクに推論はどれくらい効くか」を細かく分けて分析した。
4. prompting だけの限界（in-context window の長さ）を指摘し、finetuning との挙動の違いを示した。

## キーワード辞典

- **LLM（Large Language Model、大規模言語モデル）** … 大量の文章を読んで「次に来る単語」を予想するように訓練した、超巨大なコンピュータプログラム。文章を生成できる。
- **PaLM-540B** … Google が作った LLM の名前。540B は「5400 億個のつまみ（パラメータ）」という意味。
- **GPT-3** … OpenAI が作った LLM。本論文では追加実験で使用。
- **prompting（プロンプト）** … LLM に「お手本」と「問題」を文章で見せて、続きを書かせる使い方。学習し直さなくていい。
- **few-shot** … お手本を「ほんの数個（1〜6 個くらい）」しか見せない設定。
- **in-context example / in-context learning** … プロンプトに混ぜたお手本のこと。LLM はそれを見てパターンを真似る。
- **Chain-of-Thought（CoT、考えの連鎖）** … LLM に「順に考えを書き出してから答えて」と頼む方法。
- **Self-Consistency（CoT-SC）** … CoT を何回も（21 回など）やって、多数決で答えを決める方法。
- **hallucination（ハルシネーション）** … AI が事実じゃないことを自信満々で言ってしまう現象。
- **action space（行動空間）** … エージェントが取れる行動の選択肢の集まり。
- **thought（考え／思考の痕跡）** … この論文のキモ。LLM が出力する「独り言」のこと。環境を変えない行動として扱う。
- **trajectory（軌跡）** … 「観察 → 考え → 行動 → 観察 → …」というやりとりの記録 1 セット。
- **HotpotQA** … 複数の Wikipedia ページをまたいで答える必要のある質問応答データセット。
- **FEVER** … Wikipedia を根拠に「その主張は正しい / 間違い / 情報不足」を当てるデータセット。
- **ALFWorld** … テキストだけで描かれた家の中を動き回って課題を達成するゲーム環境。
- **WebShop** … 118 万件の商品が並ぶ模擬通販サイト。指示通りの商品を買うのが目的。
- **EM（Exact Match）** … 「答えが正解と完全に一致しているか」を測る指標。1 か 0 で採点して平均する。
- **success rate（成功率、SR）** … タスクを最後までやり切れた割合。
- **baseline（ベースライン）** … 比較対象の既存手法。
- **imitation learning（IL、模倣学習）** … 人間のお手本動作を真似るように学習する方法。
- **reinforcement learning（RL、強化学習）** … 「うまくいったら高得点」のルールで、試行錯誤して学習する方法。
- **BUTLER** … ALFWorld 用に作られた既存エージェント（IL 系）。
- **finetuning（追加学習）** … 既存の LLM に、特定タスク向けの追加データで学び直しさせること。
- **frozen（凍結）** … 追加学習せず、つまみを固定したまま使うこと。
- **greedy decoding（貪欲生成）** … LLM が単語を生成するとき、毎回「いちばん確率の高い 1 語」だけを選ぶやり方。
- **beam search** … 候補を複数並行で探すデコーディング戦略。論文中で「将来の改善案」として言及。
- **ReAct-IM** … ReAct の比較対照版。Inner Monologue 風に「考え」を「今の状態の言い換え」だけに制限した、効果検証用の改造版。
- **Inner Monologue（IM）** … 先行研究の名前。ロボットに「今の状況を独り言で言わせる」研究。
- **action**（行動） … エージェントが世界に対して取る操作。例：`search[X]`、`go to coffeetable 1`。
- **observation**（観察） … 行動の結果として環境から返ってくる情報。例：検索結果のテキスト。

## ちょっと深掘り（中学生は飛ばして OK）

- **「考えを行動に混ぜる」だけで本当にそんなに変わるの？** 著者は ALFWorld で「ReAct-IM」という対照実験をしている。これは「考え」を出させるんだけど、その考えを **今見えている状態を言い換えるだけ** に制限した版。結果は ReAct 71 vs. ReAct-IM 53。つまり「ただ独り言を言う」じゃなく「**ゴールを分解する／常識を持ち込む／例外に対処する**」という抽象的な推論が入って初めて意味がある、ということ。
- **ハイブリッドが最強な理由**。HotpotQA で ReAct 単独は CoT に少し負けている。著者の解釈は「ReAct は外の情報に頼りすぎて自分の知識を活かしきれない、CoT は自分の頭だけで突っ走ってハルシネーションする」。だから「**多数決で答えがバラついたら（=自信ない）外を調べる**」「**外で答えが見つからなかったら自分の頭で考える**」というスイッチが効く。
- **prompting だと弱いのに finetuning だと強い**。ReAct は「考え + 行動 + 観察」と書くものが多いので、お手本 6 個ではモデルが真似きれないことがある。でも 3,000 件の trajectory で追加学習すると、PaLM-8B（小さい）でも PaLM-540B prompting（大きいけど見本だけ）を全部抜く。これは「**お手本を 6 個だけ見せる方法に限界がある**」ことを示している。
- **失敗のパターン**。ReAct の最頻失敗は「同じ考えと行動を繰り返してループする」（reasoning error 47% の中の大きな部分）。著者は **greedy decoding（毎回いちばん確率の高い単語だけを選ぶ）** のせいだと推測している。beam search などの別のデコーディングや、何度かサンプリングして多数決する仕組みで改善できる余地が大きい。
- **Wikipedia API の作りはわざと弱くしている**。`search` / `lookup` / `finish` の 3 種類だけで、ページ名の完全一致でしか引けない。これは「最先端の検索エンジンを使えば当然強くなる」けど、論文の主張は **「LLM がツールを使いこなす素の力」を見たい**、というもの。だから人間が Wikipedia を読むのと同じ程度のツールに絞っている。
- **「supervised SoTA」とは大差**。HotpotQA で ReAct→CoT-SC 35.1 に対し、専用学習システムは 67.5。著者は「prompting だけで戦うのは無理。finetuning に高品質な人手データを足すべき」と認めている（discussion.tex）。
