# Editing Models with Task Arithmetic（タスク算術でモデルを編集する）

- arXiv: https://arxiv.org/abs/2212.04089
- 一次ソース: ../papers/arXiv-2212.04089v3/
- 正規ノート: ../notes/arXiv-2212.04089v3.md

---

## 一言で言うと

学習し終わったAIの「重み（中身の数字）」を、足したり引いたりするだけで、新しい技を覚えさせたり悪い癖を消したりできる、という発見の論文だよ。

## どんな問題を解こうとしてるの？

- AI（ニューラルネットワーク。たくさんの数字＝「重み」を組み合わせて、画像を見分けたり文章を作ったりする計算機のこと）は、最初に「事前学習」という大量の練習で基礎力をつけてから使われるよ。
- でも、いざ使う場面では「もっと交通標識を上手に見分けてほしい」「悪口を言う癖をやめてほしい」「個人情報を読み取らないようにしたい」と、後から AI の性格を変えたい場面がたくさんあるんだ。
- これまでのやり方は **追加で勉強させる（fine-tuning。ファインチューニング ← 一度作ったAIに、新しいデータでもう一回練習させて性格を寄せる作業）** がメインで、お金も時間もかかるし、やり方によっては元の力まで壊してしまうことがあった。
- もっと安く・速く・追加の勉強なしで AI を編集する統一的な方法が欲しい、というのがこの論文の出発点。

## どうやって解いたの？

### コアアイデア

AI の中身は「ものすごくたくさんの数字の集まり」だよ。例えるなら、何百万個もの音量つまみがズラリと並んだ巨大なミキサーを想像してみて。

- 事前学習が終わった状態のつまみの位置を **「基本セット」** と呼ぶ。
- ある仕事（例: 交通標識を見分ける）を覚えさせた後のつまみの位置を **「上手バージョン」** と呼ぶ。

このとき著者たちは、**「上手バージョン」のつまみ位置から「基本セット」のつまみ位置を引いた差分** を **タスクベクトル（task vector）** と呼んでいるよ。
（ベクトル ← 数を順番に並べたもの。住所の数字みたいに、いっぱいの数で 1 つの物を表す道具だよ）

このタスクベクトルがすごいのは、**ただの引き算で作っただけなのに、足したり引いたり混ぜたりすると AI の性格が予想通りに変わる** ことだよ。例えるなら、料理のレシピで「砂糖を 10g 増やす」というメモを別の料理に貼り付けたら同じく甘くなる、というイメージ。

### 仕組み

著者が提案する 3 つの算術操作はこんな感じ。

- **step1: タスクベクトルを作る** — 仕事 t を覚えさせる前と後で、つまみの位置の差を取って、これを「仕事 t のタスクベクトル」と呼ぶ。
- **step2: 引き算（negation。ネゲーション ← 「マイナスをつける」という意味）** — タスクベクトルを引き算するように貼り付けると、その仕事が **下手** になる。悪口を言う癖や、覚えてほしくない情報を忘れさせるのに使える。
- **step3: 足し算（addition）** — いろんな仕事のタスクベクトルを全部足し算で貼り付けると、**1 個のAIで複数の仕事ができる** ようになる。8 個の仕事を 1 個のAIに詰め込むこともできた。
- **step4: 類推（analogy。アナロジー ← 「A が B に対応するように、C は何に対応する？」というたとえ）** — 「A:B = C:D」という関係になる 4 つの仕事のうち 3 つだけタスクベクトルがあれば、4 つ目のタスクベクトルを「C + (B − A)」で作れる。データが無い仕事に対しても効くんだ。
- **step5: スケーリング係数 λ を調整** — 貼り付けるときには「どれくらい強く貼るか」の倍率 λ（ラムダ）を決めて掛ける。手元の「お試し用データ」で一番成績の良い λ を選ぶよ。

### 主要な数式

#### ① タスクベクトルの作り方

$$\tau_t = \theta_\text{ft}^t - \theta_\text{pre}$$

**この式が言ってること**: 「仕事 t を覚えさせた後のAIの中身」から「最初のAIの中身」を引き算するだけで、その仕事の特徴を表す道しるべ（タスクベクトル）が手に入るよ、という意味。

**記号の意味**:
- $\tau_t$（タウ・ティー）… 仕事 t のタスクベクトル。つまり「この仕事ができるようになるための調整メモ」のこと。
- $\theta_\text{ft}^t$（シータ・エフティー）… 仕事 t を追加で覚えさせた後の、AI の中身（何百万個もの数字の集まり）。"ft" は fine-tuned（追加練習済み）の略。
- $\theta_\text{pre}$（シータ・プリ）… 最初の事前学習だけ終わった状態の AI の中身。"pre" は pre-trained（事前学習済み）の略。
- 引き算は **要素ごと（element-wise）** に行う。つまり「同じ場所のつまみ同士で引き算する」ということ。

**身近な例え**: 料理のレシピノートを思い浮かべて。"基本のカレー" のレシピと "甘口カレー" のレシピがあったとき、両者を見比べて「砂糖 +10g、唐辛子 −2g、…」と差分だけメモしたものがタスクベクトル。このメモがあれば、別の鍋でも「甘口にする調整」だけを再利用できるよね。

#### ② タスクベクトルの適用

$$\theta_\text{new} = \theta + \lambda \, \tau_\text{new}$$

**この式が言ってること**: 元のAI の中身に、タスクベクトルを λ 倍だけ足し算して貼り付けると、新しいAI ができるよ、という意味。λ を変えれば「ちょっとだけ貼る」「強めに貼る」を調整できる。

**記号の意味**:
- $\theta_\text{new}$ … 編集後の、新しいAI の中身。
- $\theta$ … 編集する前のAI の中身（普通は事前学習済みのもの）。
- $\lambda$（ラムダ）… 「どれくらい強く貼り付けるか」の倍率。0 だと何も変わらない、1 だと普通に足す、大きいほど強く反映される。お試し用データで一番成績が良い値を選ぶ。
- $\tau_\text{new}$ … 貼り付けたいタスクベクトル（次の式①〜③のどれかで作る）。

**身近な例え**: 味噌汁を作るとき、出汁の素を入れる量を「ティースプーン何杯にするか」決めるのと同じ。少なすぎると味が出ないし、入れすぎるとしょっぱい。一番おいしい量を試しながら決める感覚が λ。

#### ③ 引き算：忘れさせる

$$\tau_\text{new} = -\tau$$

**この式が言ってること**: タスクベクトルにマイナスをつけて貼り付けると、その仕事が下手になる（＝忘れる、抑える）方向にAI を動かせる。

**記号の意味**:
- $\tau$ … 忘れさせたい仕事のタスクベクトル。例: 「悪口を上手に言う」タスクベクトル。
- $-\tau$ … その逆向き。「悪口を言う方向と反対」にAI を動かす道しるべ。
- これを上の式②に入れて $\theta_\text{new} = \theta - \lambda\,\tau$ として使う。

**身近な例え**: ジャンケンで「グーを出すクセ」を直したいとき、わざと「グーを出したくなる気持ちを反対向きに引っ張る」ようなもの。クセそのものを抑える方向に体を動かす感じ。

#### ④ 類推（アナロジー）

$$\tau_\text{new} = \tau_C + (\tau_B - \tau_A)$$

**この式が言ってること**: 「A:B = C:D」という関係（A から B に変えるのと同じ変え方を C にも適用すると D になる）のとき、A・B・C のタスクベクトルだけ持っていれば、D のタスクベクトルを作れる。D のデータが手元に無くてもいい。

**記号の意味**:
- $\tau_A, \tau_B, \tau_C$ … 仕事 A, B, C それぞれのタスクベクトル。
- $\tau_B - \tau_A$ … 「A を B に変えるための差分」。例: 「Amazon レビューで言語モデル」→「Amazon レビューで感情判定」に変える差分。
- $\tau_C + (\tau_B - \tau_A)$ … その「変え方」を C に適用した結果。「Yelp レビューで言語モデル」に同じ差分を足せば、「Yelp レビューで感情判定」ができるはず、というアイデア。

**身近な例え**: 地図のたとえが分かりやすいよ。「東京駅から渋谷駅へ行く道のり」を知っていれば、「大阪駅」から同じ向き・同じ距離だけ歩けば、大阪での "渋谷っぽい場所" にたどり着けるかも、という発想。実際の街なら誤差は出るけど、AI の重み空間ではちゃんと近い場所に着く、というのが面白い結果。

## 何がすごいの？

著者が論文で示している主な結果と「貢献」は次の通り（数字は全部 TeX から）。

- **画像で「忘れさせる」**（Table 1、CLIP ViT-L/14 ← CLIP は画像と文章をペアで学習した AI、ViT-L/14 はそのサイズ違い）
  - 8 つの画像仕事の平均正答率: 事前学習だけのとき **64.8%** → タスクベクトルを引き算した後 **19.0%**（−45.8 ポイントの「忘却」に成功）。
  - 一方、関係ない仕事（ImageNet ← 画像分類の有名なベンチマーク）の正答率は 75.5% → 72.9%（−2.6 ポイントだけ）。
  - 比較した他の「忘却法」: gradient ascent（勾配上昇法 ← AI の学習を「わざと逆向き」にする方法。詳しくは次節）は target 3.93% だが ImageNet が 16.3% まで崩壊。ランダムなベクトルを引いただけでは forget できず 60.9%。**負タスクベクトルが一番バランスが良い**。

- **文章で「悪口を言う癖を消す」**（Table 2、GPT-2 Large ← OpenAI の文章生成AI）
  - 悪口を含む文の生成率: 4.8% → **0.8%（6 倍減）**。
  - 平均「悪さスコア」: 0.06 → 0.01。
  - 関係ない文章タスク（WikiText-103 で perplexity ← どれだけ次の単語を当てられるかの指標、小さいほど良い）は 16.4 → 16.9（+0.5 だけ悪化、ほぼ同じ）。
  - 比較: gradient ascent は perplexity が 10$^{10}$（10兆）を超えて完全に壊れる。「悪口ではない文章で fine-tune する」やり方より、negation の方が悪口は減らせるし文章力も保てる。

- **足し算でマルチタスク化**（§4.1、CLIP）
  - 8 個の画像仕事のうち 2 個分のタスクベクトルを足したら、専門化された 2 つの fine-tune モデルを使ったときの **98.9%** の精度（正規化精度）を、たった 1 個のモデルで達成。
  - 8 個全部足したときの平均正規化精度は **91.2%**。

- **足し算で単独仕事も伸びる**（Table 3、T5-base + Hugging Face Hub の 427 個のチェックポイント）
  - GLUE という英語ベンチマークの 4 タスク平均: fine-tune 単独 **78.1** → タスクベクトルを足すと **78.6**（+0.5）。
  - 内訳: MRPC +0.8、RTE +0.2、CoLA +0.7、SST-2 +0.2。全部わずかに改善。

- **類推（アナロジー）で「ラベルなしのまま」精度UP**（Table 4、T5-large）
  - Yelp 感情判定の正答率: 仲間の Amazon で fine-tune しただけ 95.0 → 類推で **95.1**（ターゲットで直接 fine-tune した 95.5 に肉薄）。
  - Amazon 感情判定: 94.8 → **95.2**（ターゲットの 95.5 に肉薄）。
  - **target のラベルを使わずに**、ターゲットで直接学習したときの精度に近づけたのがポイント。

- **少数派の画像も類推で +3.4 ポイント改善**（Fig. 4）
  - 「実写の犬 / 実写のライオン / スケッチの犬」の 3 つのタスクベクトルから「スケッチのライオン」のタスクベクトルを作ると、事前学習だけのモデルより平均 +3.4 ポイント向上。これは新しい学習データを約 100 サンプル集めるのと同じくらい効くらしい。

- **著者がはっきり主張している「貢献」**:
  1. 重み空間での「タスクベクトル算術」という統一的な編集枠組み（引き算・足し算・類推の 3 つ）を提案したこと。
  2. 画像（CLIP）・自然言語（T5）・文章生成（GPT-2）の 3 種類のAI、複数のサイズで効くことを示したこと。
  3. **追加学習なし・推論時の余計な計算なし** で、マルチタスク化・忘却・データなし汎化を実現するレシピを示したこと。
  4. 世の中に公開されている学習済みモデル（Hugging Face Hub など）が、実は「タスクベクトルの倉庫」として使い回せる、と示したこと。

## キーワード辞典

出てきた順番で並べているよ。

- **事前学習（pre-training）** … AI に大量のデータで「基礎の勉強」をさせる最初の工程。ここで AI は世の中の一般知識を覚える。
- **fine-tuning（ファインチューニング）** … 一度作った AI に、もう少しだけ追加データで勉強させて、特定の仕事ができるように調整する作業。「追加練習」。
- **重み（weights）** … AI の中身そのもの。たくさんの数字（つまみ）の集合体で、これが変わるとAI の答えも変わる。
- **タスクベクトル（task vector）** … 「fine-tuning した後の重み」から「事前学習だけの重み」を引いた差分。仕事の特徴を表す「調整メモ」。
- **negation（ネゲーション、引き算）** … タスクベクトルにマイナスをつけて使う操作。AI にその仕事を「忘れさせる」効果。
- **addition（アディション、足し算）** … 複数のタスクベクトルを足して使う操作。AI に複数の仕事を同時にできる力を持たせる。
- **analogy（アナロジー、類推）** … 「A:B = C:D」という関係を使って、データが無い仕事のタスクベクトルを作る操作。
- **CLIP** … OpenAI が作った、画像と文章を結びつけて学習した AI のシリーズ。本論文の画像実験のベース。
- **ViT-B/32, ViT-B/16, ViT-L/14** … CLIP のサイズ違い。「L」は大きい（Large）、「B」は中くらい（Base）。数字は画像を小分けにする粒度。
- **GPT-2** … OpenAI が作った文章生成 AI のシリーズ。Large は大きいバージョン。
- **T5** … Google が作った「文章を入れると別の文章を出す」AI。base・small・large などサイズ違いがある。
- **GLUE** … 英語の言語理解能力を測るための、4〜10 個の小さなテストを集めたベンチマーク（成績表セット）。
- **MRPC, RTE, CoLA, SST-2** … GLUE に含まれる個別の小テスト名。文の言い換え判定や感情判定など。
- **perplexity（パープレキシティ）** … 文章生成AI の「次の単語をどれだけ当てられるか」の指標。小さいほどよく当てられる。
- **gradient ascent（勾配上昇法）** … 通常の学習はミスを減らす向きに調整するけれど、これは「わざとミスを増やす向き」に調整する忘却テクニック。
- **Detoxify** … 文章がどれくらい「毒（悪口など）」を含むかを判定する別のAI。本論文で文章の毒性を測るのに使った。
- **Civil Comments** … ネットのコメントに「毒度スコア」が振られたデータセット。本論文では毒度 0.8 超のコメントで fine-tune してから negation する。
- **ImageNet** … 1,000 種類の物の画像を集めた巨大データセット。本論文ではコントロール用（編集が他の力を壊していないかを確認）に使う。
- **Hugging Face Hub** … 世界中のAI 開発者が学習済みモデルを共有しているウェブサイト。本論文では 427 個の公開モデルをタスクベクトルの素材として使った。
- **要素ごと（element-wise）** … 同じ位置にある数字同士で計算するという意味。つまみが何百万個並んでいるとき、1 番のつまみ同士、2 番のつまみ同士、…と1個ずつ計算する。
- **正規化精度（normalized accuracy）** … 「専門化したモデルの精度」を 1 として、それを基準に正答率をスケールし直したもの。複数の仕事の難しさを揃えて平均するときに便利。

## ちょっと深掘り（中学生は飛ばして OK）

- **なぜ足し算でうまくいくの？**: Fig. 5 で著者は「異なる仕事のタスクベクトル同士はだいたい直交（向きがバラバラ）」と観察している。直交 ← 2 つのベクトルが「お互いに干渉しない・全然違う方向」を向いていること。仕事ごとに調整しているつまみの位置がバラバラだから、足してもケンカしないと考えられる。例外は、MNIST・SVHN・GTSRB（どれも数字認識が絡む）や EuroSAT・RESISC45（どちらも衛星写真）のような「似た仕事同士」だけ。

- **学習率に敏感**: Fig. 6（lr_ablation）では、fine-tune 時の学習率（learning rate ← 1 ステップでどれだけ重みを動かすかの大きさ）を上げすぎると、タスクベクトルの加算の方が単独の fine-tune よりも先に精度が崩れる、と書かれている。だから「他人の作ったタスクベクトルを混ぜるなら、低めの学習率で fine-tune されたものを選ぼう」と著者は推奨。

- **fine-tune の途中段階でもいい**: Fig. 7（intermediate task vectors）では、fine-tune の途中で取ったタスクベクトルが、最後まで学習し終えたタスクベクトルとほぼ同じ向きにすぐ収束する、と示されている。つまり「最後まで頑張って fine-tune しなくても、早めに切り上げてもそこそこ良いタスクベクトルが手に入る」ということ。計算コストの節約になる。

- **制限**: 著者自身が §6 Limitations ではっきり書いている通り、
  - 同じ **アーキテクチャ**（AI の設計図）でないと足し引きできない。
  - 同じ **事前学習の初期化**（スタート地点が同じ重み）から fine-tune されたモデル同士でないとダメ。
  - ただし Hugging Face Hub には「同じ BERT-base から派生したモデル」が 3,000 個以上、「同じ T5-small から派生したモデル」も 800 個以上あるから、実用上は素材に困らない、とも書いている。

- **negation は本当に「意味として」毒を消しているの？**: TeX には書かれていないが、おそらく Detoxify という別のAI による毒性スコアに頼っているだけなので、表面的なトリガー単語を避けているのか、それとも本当に毒のある考えを抑え込めているのかまでは、この論文だけでは断言できない（評価方法の限界として考えられる）。
