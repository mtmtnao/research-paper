# Generative Adversarial Nets（敵対する 2 つの AI を競わせて画像を作る仕組み）

- arXiv: https://arxiv.org/abs/1406.2661
- 一次ソース: ../papers/arXiv-1406.2661v1/
- 正規ノート: ../notes/arXiv-1406.2661v1.md

---

## 一言で言うと

「偽物を作る AI」と「偽物を見抜く AI」を本気でケンカさせて、最終的に本物そっくりの画像が作れるようにする方法を考えたよ。

## どんな問題を解こうとしてるの？

- やりたいこと: コンピュータに「人の顔の写真」とか「手書き数字の画像」みたいな、データを **新しく作り出す**（生成する）能力を持たせたかった。これを「生成モデル」と呼ぶよ（← データを新しく作る AI のこと）。
- 困りごと: それまでの「生成モデル」は、計算がものすごく難しい確率の計算（partition function ＝ 全パターンの確率の合計を出す作業）をやらないと学習できなかった。これは、たとえば「サイコロを 100 個振った時のあらゆる出目の組み合わせ」を全部数え上げるみたいな、現実的にはムリな作業。
- だから生成 AI は、「写真を見て犬か猫か当てる」みたいな見分ける AI（判別モデル）に比べて、ぜんぜん発展してこなかった。
- 既存の方法（RBM, DBM, DBN というやつら ← 名前は覚えなくて OK。昔の生成モデルの仲間たち）は、MCMC というサイコロを何回も振って近似する技を使うんだけど、これがうまく動かないことが多かった。

## どうやって解いたの？

### コアアイデア

著者は「**ニセ札を作る犯人と、それを見抜く警察を、同じ街で同時に成長させたらどうなる？**」というたとえを出してる。

- 最初、犯人（生成器 G）はヘタクソなニセ札しか作れない。警察（判別器 D）は本物との違いを一目で見抜ける。
- 犯人は「どうすれば警察にバレないか？」を学んでだんだん上達する。
- 警察も「どうすればウソを見抜けるか？」を学んでだんだん鋭くなる。
- これを延々と繰り返すと、最後には犯人のニセ札が本物と区別できないレベルになる。

これが GAN（Generative Adversarial Nets ＝ 敵対的生成ネットワーク）の全体像。「敵対的（adversarial）」というのは、お互いに敵として戦わせるという意味。

### 仕組み

- step1: **生成器 G** を用意する。これは「ランダムなノイズ（サイコロを振って出した適当な数の列）」を受け取って、画像っぽいデータを吐き出す機械。中身は MLP（多層パーセプトロン ← 数字を入れたら別の数字を出す計算装置を何段も重ねたもの。ニューラルネットの一番シンプルな形）。
- step2: **判別器 D** を用意する。これは画像を受け取って「これは本物の写真っぽい？それとも G が作ったニセモノっぽい？」を 0〜1 の数字（1 に近いほど本物っぽい）で答える機械。これも MLP。
- step3: 本物の画像と、G が作ったニセ画像を両方 D に見せて、「本物には 1、ニセモノには 0 を出す」ように D を学習させる（D が上手になる）。
- step4: G は「D が間違えてニセ画像を本物だと言ってしまう」ように学習する（G が上手になる）。
- step5: step3 と step4 を交互にくり返す。論文では「D を $k=1$ 回更新するごとに G を 1 回更新」している。
- step6: ノイズは G の一番下の層にだけ入れる（中間層に入れてもいいけど実験ではやってない、と TeX に書かれている）。

学習には backpropagation（バックプロパゲーション ← 答えとの間違いを少しずつネットワーク全体に伝えてパラメータを直す方法）と dropout（ドロップアウト ← 学習中わざと一部の計算をオフにして、ズルしないようにする工夫）だけを使う。MCMC（モンテカルロという、サイコロを何回も振って近似する重い計算）は一切いらない。

### 主要な数式

#### ① ゲームのルール（G と D が何を競っているかを表す式）

$$\min_G \max_D V(D, G) = \mathbb{E}_{\bm{x} \sim p_{\text{data}}}[\log D(\bm{x})] + \mathbb{E}_{\bm{z} \sim p_{\bm{z}}}[\log (1 - D(G(\bm{z})))]$$

**この式が言ってること**: 「警察 D は、本物には『本物！』と、ニセモノには『ニセモノ！』と自信を持って言えるように **最大化**する。犯人 G は逆に、D に『これ本物だわ』と勘違いさせるように **最小化**する」というルール。両者がお互い逆向きに頑張るゲームを式にしたもの。

**記号の意味**:
- $G$ … 生成器（犯人）。ノイズから画像っぽいものを作る機械。
- $D$ … 判別器（警察）。画像を見て本物っぽさを 0〜1 の数字で答える機械。
- $\bm{x}$ … 本物のデータ（写真など）。$\bm{}$ は「ベクトル＝数字を順番にずらっと並べたもの。たとえば写真は色の数字を全部並べたら 1 枚の絵を表せる」を強調する記号。
- $\bm{z}$ … ランダムなノイズ（サイコロで出した適当な数字の列）。
- $p_{\text{data}}$ … 本物のデータが従う「確率分布」（← どの画像がどれくらいの割合で出てくるかのルール）。
- $p_{\bm{z}}$ … ノイズの分布（たとえば 0〜1 で均等に出るとか）。
- $G(\bm{z})$ … 犯人 G がノイズ $\bm{z}$ から作ったニセ画像。
- $D(\bm{x})$ … D が画像 $\bm{x}$ を見て「本物っぽさ」を返した数（0〜1）。
- $\log$ … 対数。ここでは「D の答えが 1 に近いほど 0 に、0 に近いほどマイナス方向に大きくなる」変換だと思えば OK。スコア板みたいなもの。
- $\mathbb{E}[\dots]$ … 期待値（「いろんなデータでこのスコアの平均を取ったらいくつ？」という意味）。テストを何回も受けて平均点を出すのと同じ。
- $\min_G \max_D$ … 「G は小さくしたい、D は大きくしたい」というケンカの方向を表す記号。

**身近な例え**: ジャンケン大会で「あいこになった回数」をスコアとして、片方のチームはスコアを増やしたい、もう片方は減らしたい、という綱引きのようなゲーム。両方が同時にうまくなることで、最終的にどちらも実力 MAX になる。

#### ② 最強の警察はどんな判断をするか

$$D^*_G(\bm{x}) = \frac{p_\text{data}(\bm{x})}{p_\text{data}(\bm{x}) + p_g(\bm{x})}$$

**この式が言ってること**: 「ある画像 $\bm{x}$ を見たとき、最強の警察は『本物が来る確率』÷『本物が来る確率＋ニセモノが来る確率』を答えとして返すのが理論的にベスト」ということ。つまり、本物の出現割合がニセモノより多い画像には高いスコアを、ニセモノが多い画像には低いスコアをつける。

**記号の意味**:
- $D^*_G$ … 「G が今のままだとして、最高に賢い警察 D」のこと。$*$ は「最強の」の印。
- $\bm{x}$ … 判定したい画像。
- $p_\text{data}(\bm{x})$ … その画像 $\bm{x}$ が本物のデータの中で出てくる割合。
- $p_g(\bm{x})$ … その画像 $\bm{x}$ が G の作るニセモノの中で出てくる割合。

**身近な例え**: スーパーで「リンゴ売り場のリンゴ」と「お祭り屋台のオモチャのリンゴ」が混ざってるとき、その物が「本物の割合 ÷（本物の割合＋ニセモノの割合）」を答えれば、それが一番正解に近いリンゴ度になる。最強の鑑定士の答え方そのもの。

#### ③ ゲームが完璧に決着したときに行きつく場所

$$C(G) = -\log(4) + 2 \cdot \text{JSD}(p_\text{data} \| p_g)$$

**この式が言ってること**: 警察を毎回ベストにした状態で犯人を評価したスコアは、「定数 $-\log 4$」＋「2 ×（本物の分布と G の分布の **ズレ具合**）」という形になる。だから G が本物の分布にぴったり一致した時（ズレ＝0）にスコアは最小値 $-\log 4$ となり、それ以外では必ずもっと大きくなる。つまり「ぴったり真似する」のがゲームの唯一の最適解だと数学的に証明できる。

**記号の意味**:
- $C(G)$ … G を評価する点数（小さいほど G が上手）。
- $-\log 4$ … 定数。$\log 4 \approx 1.39$ なので $-\log 4 \approx -1.39$ という固定の数字。ゲームの「最低点」。
- $\text{JSD}(p_\text{data} \| p_g)$ … Jensen–Shannon ダイバージェンス。2 つの分布のズレ具合を測る道具。**完全に一致するなら 0、ズレてればプラスの数になる**（マイナスにはならない）。
- $p_\text{data}$ … 本物データの分布。
- $p_g$ … G が作るニセ画像の分布。

**身近な例え**: 「2 人のテストの答案がどれだけ似てるか」を 0 以上の点数で測る方法があって、ピッタリ同じなら 0、違うほど大きくなる。ゲームの最終スコアは「もとからある最低点」＋「この答案の違い × 2」になるので、答案を完全に真似するのが満点（＝最低スコア）への唯一の道。

## 何がすごいの？

- **理論的な保証**: 非パラメトリック（G と D が十分に賢いと仮定した理想状態）の証明で、「ゲームの唯一の最適解は『G が本物の分布を完全再現する』こと」と数学的に示した。これは生成モデルにはとても珍しい強い性質。
- **MCMC が不要**: 学習にも、画像を作るときにも、サイコロ振りまくり系の重い計算（Markov chain）が要らない。backprop と dropout だけで動く。
- **Parzen window 評価（Table 1）**: 生成した画像の「もっともらしさ」を別の方法で点数化したスコア。大きいほど良い。
  - MNIST（手書き数字データセット）: Adversarial nets が **225 ± 2**（最高スコア）。比較対象は DBN: 138 ± 2、Stacked CAE: 121 ± 1.6、Deep GSN: 214 ± 1.1。
  - TFD（Toronto Face Database ＝ 顔写真データセット）: Adversarial nets が **2057 ± 26**（Stacked CAE の 2110 ± 50 に次いで 2 位）。DBN: 1909 ± 66、Deep GSN: 1890 ± 29。
- **CIFAR-10（10 種類の物体の小さい写真データセット）**: 全結合版と、convolutional（畳み込み ← 画像処理で近所のピクセルをまとめて見る仕組み）版の両方で画像生成に成功。数字での比較はせず、サンプル画像のみ提示。
- **記憶じゃないことを示した**: 生成サンプルの隣に「学習データの中で一番似ているもの」を並べて、丸暗記してないことを Figure 2 で見せた。
- **滑らかに変化**: ノイズ $\bm{z}$ を少しずつ変えると、生成される画像も滑らかに変化することを Figure 3 で示した。
- **著者が論文で挙げた貢献（Conclusions）**:
  1. 2 人プレイヤーの敵対ゲームで生成モデルを学習する一般的な枠組みを提案したこと。
  2. 理論的に「最適解は本物と一致する」ことを証明したこと。
  3. MLP ＋ backprop ＋ dropout だけで動く実装方式。
  4. 既存の深層生成モデルとの比較表（Table 2）。

## キーワード辞典

- **生成モデル（generative model）** … データを新しく作り出せる AI。
- **判別モデル（discriminative model）** … データを見分ける AI（猫か犬か当てるとか）。
- **生成器 G（generator）** … GAN でニセモノを作る側。
- **判別器 D（discriminator）** … GAN で本物かニセモノかを当てる側。
- **MLP（multilayer perceptron）** … 多層パーセプトロン。一番シンプルな構造のニューラルネット。
- **ニューラルネット** … 入力（数字の列）を何段もの計算で別の数字に変える機械。脳の神経をマネしてる。
- **backpropagation（バックプロパゲーション、誤差逆伝播）** … 答えとのズレを後ろから前へ伝えて、各パラメータを少しずつ正しい向きに直す方法。
- **dropout** … 学習中、一部の計算をわざと無効にする工夫。丸暗記を防ぐ。
- **MCMC（Markov chain Monte Carlo）** … サイコロを何回も振って答えを近似する手法。重い。
- **partition function（分配関数）** … 全パターンの確率の合計を計算する処理。たいてい現実的にはムリ。
- **確率分布** … 「どの値がどれくらいの割合で出るか」のルール。
- **期待値 $\mathbb{E}$** … たくさん試した平均値のこと。
- **対数 $\log$** … 0 に近づくほどマイナス方向に大きくなる数学関数。スコアの圧縮みたいな働き。
- **minimax ゲーム** … 「相手のスコアを下げる × 自分のスコアを上げる」を同時にやる戦い。
- **JSD（Jensen–Shannon divergence）** … 2 つの確率分布がどれだけ違うかを測る数。同じなら 0、違うほど大きい。
- **KL ダイバージェンス（Kullback–Leibler divergence）** … JSD の親戚で、これも 2 つの分布のズレを測る道具。
- **Parzen window** … 生成サンプルから「画像のもっともらしさ」を推定する評価手法。
- **MNIST** … 手書き数字 0〜9 の画像データセット。生成モデルの定番ベンチマーク。
- **TFD（Toronto Face Database）** … 顔写真のデータセット。
- **CIFAR-10** … 10 種類の物体の小さなカラー写真データセット。
- **DBN / DBM / RBM** … GAN 以前の生成モデルの仲間たち（Deep Belief Net / Deep Boltzmann Machine / Restricted Boltzmann Machine）。比較対象。
- **GSN（Generative Stochastic Network）** … 別系統の生成モデル。Table 1 の比較対象。
- **maxout** … 判別器に使われた活性化関数（ニューラルネットの中の小さな計算部品）の一種。
- **rectifier linear / sigmoid** … 生成器で使われた活性化関数の種類。
- **Helvetica scenario（モード崩壊）** … G がサボって「どんなノイズを入れても同じ画像しか作らなくなる」失敗パターン。著者自身が論文で警告している。

## ちょっと深掘り（中学生は飛ばして OK）

- **G が本物データを直接見ない**: G は本物の画像を直接受け取らず、「D を通った勾配」だけで学習する。だから「学習データをそのままコピーしてズルする」というルートが構造的に塞がれている。著者はこれを **統計的なメリット**として強調している。
- **non-saturating loss という実装上のテクニック**: 学習初期に G がヘタクソだと、$\log(1 - D(G(\bm{z})))$ が飽和（saturate ← 値がほぼ動かなくなる）してしまい、G がうまく学習できない。だから論文では「$\log(1 - D(G(\bm{z})))$ を最小化する」代わりに「$\log D(G(\bm{z}))$ を最大化する」という同じ最適点を持つけど勾配の強い目的に切り替える、と書かれている。後の GAN 研究で「non-saturating loss」と呼ばれる定番テク。
- **収束の理論と現実のギャップ**: 「収束する」と証明したのは $p_g$ という確率分布の空間での話で、実際に動かしている MLP のパラメータ空間ではそうはならない（複数の臨界点が出る）。著者自身も「でも実用的には MLP はちゃんと動くからまぁいいよね」と希望的観測で片づけている（§4.2 末尾）。実務での不安定さの理論的な保証は無い。
- **$k=1$ の理由**: D を $k$ 回、G を 1 回交互に学習するが、論文では「$k=1$ は最も計算が安い選択肢だから使った」とだけ書かれている。それで安定するかどうかの定量的議論はない。
- **限界として著者自身が挙げているもの**:
  - $p_g(\bm{x})$ の **明示的な表現が無い**（「この画像が出てくる確率はいくつ？」と聞かれても答えられない、サンプルは出せるけど）。
  - D と G の同期を保たないと **Helvetica scenario**（モード崩壊）になる。
  - Parzen window 評価は高次元では信頼性が低い。
- **CIFAR-10 の定量比較は無い**: Table 1 は MNIST と TFD だけ。CIFAR はサンプル画像を見せるだけで、「数値で勝った」とは言っていない。
